Tag 和 Category 的不同

在写静态博客的时候，front matter 中可以指定 Category 和 Tag，由于一些静态博客主题的 Category 页面和 Tag 页面是一样的，所以我对这两个概念一直感到很迷惑。 去搜了搜，发现知乎有个问题：Tag、Label、Category 三者的区别是什么，下面这个回答挺不错的： Tag，标签，一般包含一段内容的属性，可以是分类，可以是话题，也可以是作者，地理信息等。 Label，标记，是网页框架中一个区域的功能概括。比如知乎首页上的“浏览”、“问题”、“通知”就是label。选择不同的标记，相应区域的功能就会发生变化。 Category，分类，可以理解为种属概念。一段内容有且只有一个种属的分类。和 Tag 不同的是一段内容可以有多个 tag，并且 tag 是未预先设定的，一段内容可以有多个 Tag，也可以没有 Tag。但 caterory 在一个分类标准下只有一个。 作者：郑迅 链接：https://www.zhihu.com/question/19561626/answer/12227503 来源：知乎 著作权归作者所有。商业转载请联系作者获得授权，非商业转载请注明出处。 关键在于：Category 已经先于内容定好，每个内容发布时必须有一个 Category，而 Tag 可以发布时指定，也可以不指定，同时后续也可以添加。 Bilibili 里分区（Category）和标签（Tag）的逻辑就可以很好解释 Category 和 Tag 的区别，在 Bilibili 中，发布视频时，我们必须指定某个分区才能发布，而标签则不需指定。视频发布后，可以添加和修改 Tag。 如上图，分区是必选且只选其一的，而标签可以创建也可以不创建 早期 Bilibili 是可以给别人的视频添加标签的，但是由于有人恶意加标签就取消了这个功能，但自己还是可以给自己的视频修改或者添加标签，而分区则是不能修改的：
